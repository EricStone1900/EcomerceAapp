# 麦克风权限文案说明

> 语音搜索功能需要用户授权麦克风访问。本文档记录了权限文案，供提审时同步给相关同学。

## Info.plist 配置

已在 Xcode 项目 build settings 中配置 `INFOPLIST_KEY_NSMicrophoneUsageDescription`：

```xml
<key>NSMicrophoneUsageDescription</key>
<string>语音搜索功能需要访问您的麦克风</string>
```

## 文案说明

| 语言 | 文案 | 用途 |
|------|------|------|
| 简体中文 | 语音搜索功能需要访问您的麦克风 | 首次请求权限时的系统弹窗 |

## 交互流程

1. 用户点击语音搜索按钮
2. 系统弹出麦克风权限授权框（显示上述文案）
3. 用户选择：
   - **允许** → 进入"聆听中"状态
   - **拒绝** → 展示引导提示"麦克风权限被拒绝，请前往设置开启后使用语音搜索"，可点击"去设置"跳转
4. 如果用户在设置中更改权限后返回 App，语音搜索按钮恢复正常可用

## App Store 审核注意事项

- 该权限仅在你使用语音搜索功能时触发请求
- 非强制授权：拒绝权限后不影响正常浏览商品和购物车功能
- 有完善的降级体验：拒绝后不会崩溃或无限弹窗
